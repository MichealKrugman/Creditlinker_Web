Financer Portal — API Key Management UI
The core idea to keep in mind:
A financer is not a developer by default. They become one by opting in. So the UI lives in the financer settings, not a separate developer portal. The flow is:

Financer visits Settings → Integrations (or API Access)
They see a locked state — "Connect your systems to CreditLinker via API" with a button to enable API access
On enabling, a developer_accounts record is created behind the scenes with account_type = 'institutional' and institution_id linked to their institution — they never see or think about this
They can now generate API keys — same UI as the standalone developer dashboard but scoped to institutional endpoints only
Keys are prefixed sk_live_ / sk_test_ same as standalone developers
The key management page shows only institution-scoped endpoints in the docs — institution-query, discovery-match, confirm-settlement — not partner endpoints

The key constraint: one institution = one developer account. When they generate keys, those keys inherit institution_id automatically. They can never query another institution's data.
